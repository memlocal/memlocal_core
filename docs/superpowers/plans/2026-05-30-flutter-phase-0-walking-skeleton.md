# memlocal Flutter — Phase 0: Walking Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the `memlocal` Flutter package skeleton and prove, on a real Android device/emulator and an iOS simulator, that the CozoDB-backed Rust engine opens over FFI and that Rust can call back into a Dart closure — burning down the two highest project risks before any breadth work.

**Architecture:** A Flutter FFI plugin package at `memlocal_core/flutter/` (pub name `memlocal`) wraps a Rust crate `memlocal_flutter` at `flutter/rust/`, which depends on the existing `memlocal_core` via a path dependency. flutter_rust_bridge (FRB) v2.11.1 generates the Dart↔Rust glue; cargokit compiles the Rust during `flutter build`. Phase 0 exposes only `Memlocal.openInMemory()`, `memoryCount()`, and a `callDartClosure()` callback probe.

**Tech Stack:** Rust 1.94.1, flutter_rust_bridge `=2.11.1`, cargokit, Flutter 3.41.5, CozoDB (SQLite backend, via memlocal_core), Android NDK, Xcode.

**Reference spec:** `docs/superpowers/specs/2026-05-30-memlocal-flutter-package-design.md` (§5 architecture, §6 bridge, §14 phases, §15 risks). This is the first of the per-phase plans; Phases 1–5 each get their own plan once this one lands.

**Working branch:** `flutter-package` (already created). All commits land here.

- [ ] **Step 1: Write the Rust FFI + a failing host unit test**

Create `flutter/rust/src/api/skeleton.rs`:
```rust
use std::sync::Arc;

use memlocal_core::api::MemlocalEngine;
use memlocal_core::models::{CoreConfig, StorageConfig};

/// Opaque handle held by Dart. (FRB treats unknown structs as opaque.)
pub struct Memlocal {
    pub(crate) engine: Arc<MemlocalEngine>,
}

impl Memlocal {
    /// Open an in-memory engine. Phase 0 smoke entry point.
    pub fn open_in_memory(dimensions: u32) -> Result<Memlocal, String> {
        let config = CoreConfig {
            storage: StorageConfig {
                in_memory: true,
                embedding_dimensions: dimensions,
                ..Default::default()
            },
            ..Default::default()
        };
        let engine = MemlocalEngine::open(config).map_err(|e| e.to_string())?;
        Ok(Memlocal { engine: Arc::new(engine) })
    }

    /// Total stored memories (None = all types).
    pub fn memory_count(&self) -> Result<u32, String> {
        self.engine
            .memory_count(None)
            .map(|c| c as u32)
            .map_err(|e| e.to_string())
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn opens_in_memory_and_counts_zero() {
        let m = Memlocal::open_in_memory(1536).expect("engine opens");
        assert_eq!(m.memory_count().expect("count works"), 0);
    }
}
```

use flutter_rust_bridge::DartFnFuture;

/// Calls a Dart-provided async closure and returns its result.
/// Proves FRB can invoke Dart back from Rust (foundation for Dart-side providers).

    value: i32,
