# Phase 0 — Discoveries (memlocal Flutter walking skeleton)

Date: 2026-05-30 · Branch: `flutter-package`

**Outcome: the two highest project risks are retired.** The CozoDB-backed Rust engine cross-compiles for both Android and iOS, FRB opaque handles work with zero friction, and the Dart↔Rust callback foundation compiles on both platforms. On-device *runtime* verification is deferred pending local emulator/simulator setup (does not affect viability).

## Toolchain (pinned / observed)
- `flutter_rust_bridge_codegen`: **2.11.1** (matches core's `=2.11.1` pin)
- Rust 1.94.1 · Flutter 3.41.5 (Dart 3.11.3) · Xcode 26.5 · CocoaPods 1.16.2
- Rust targets installed: `aarch64-apple-darwin`, `aarch64-apple-ios`, `aarch64-apple-ios-sim`, `x86_64-apple-ios`, `aarch64-linux-android`, `armv7-linux-androideabi`, `x86_64-linux-android`, **`i686-linux-android`** (last one auto-added by cargokit during the Android debug build)
- Android: SDK 36.1.0, **NDK 28.2.13676358 (r28c)** at `~/Library/Android/sdk`; JDK 21; all licenses accepted

## Cross-compile gates
- **Android: PASS.** `flutter build apk --debug --target-platform android-arm64` (from `flutter/example`) built `app-debug.apk` in ~140s; cargokit compiled `memlocal_core` (CozoDB) for arm64 + x86_64 + i686 and linked the `.so`. No NDK/linker errors. `minSdk`: Flutter default sufficed (no override needed).
- **iOS: PASS (toolchain-level).** `flutter build ios --simulator` could not run here (environment gap, below), so verified directly: `cargo build --lib --target aarch64-apple-ios-sim` (from `flutter/rust`) compiled `cozo v0.7.6` + `memlocal_core` + `memlocal_flutter` in ~37s. `lipo -info` → arm64; Mach-O `LC_BUILD_VERSION` → platform 7 (iOS Simulator), `minos 14.0`. The new callback FFI cross-compiles for both targets too.

## FRB integration
- `Memlocal { engine: Arc<MemlocalEngine> }` was accepted as an FRB opaque with **zero Send+Sync friction** — the flagged risk did not materialize.
- Exposed surface so far: `Memlocal.openInMemory({dimensions})`, `Memlocal.memoryCount()`, `callDartClosure({value, callback})`. `flutter analyze` clean throughout; host `cargo test` green (`opens_in_memory_and_counts_zero`).

## Binary size (DEBUG figures — informational; release will be far smaller)
- Android `arm64-v8a` `.so`, stripped (as packaged in APK): **~36 MB**; unstripped debug: ~189 MB
- iOS-sim `libmemlocal_flutter.a` (debug, unstripped): ~285 MB (`libmemlocal_core.a` ~265 MB)
- TODO (later phase): measure a **release**, split-per-ABI build for the real shippable size.

## Deferred / environment gaps (do NOT block viability)
- **Android runtime check:** no AVD/emulator on this machine. Once one is booted: `flutter test integration_test/simple_test.dart -d <android-id>` (asserts engine opens + `memoryCount()==0`).
- **iOS runtime check:** Xcode 26.5 ships SDK `iphonesimulator26.5` but only the **iOS 26.4** simulator runtime is installed → Xcode offers zero valid simulator destinations. Fix: `xcodebuild -downloadPlatform iOS` (multi-GB) or Xcode → Settings → Components. Then the integration test + the callback round-trip can be verified on the sim.
- **Callback round-trip:** `callDartClosure` compiles on both platforms; the live round-trip (returns `42`) is provable only on a running app → deferred with the above.

## Notes for later phases
- **Manual mobile `cargo build` needs NDK env vars** in a clean shell (e.g. `CC_aarch64_linux_android`, `CARGO_TARGET_AARCH64_LINUX_ANDROID_LINKER` → NDK clang). The *package* build (cargokit/Gradle) sets these itself, so `flutter build apk` works without them; this only affects a direct `cargo build --target aarch64-linux-android`. Consider a committed `.cargo/config.toml` or a contributor doc note (Phase 5).
- **Desktop platform dirs** (`linux/`, `macos/`, `windows/`) came from the `flutter create` default inside the FRB plugin template. Decide in Phase 5 whether to trim to mobile-only.
- **iOS pod scaffolding** (`example/ios/Runner.xcodeproj/project.pbxproj`, `Runner.xcworkspace/contents.xcworkspacedata`, `Podfile.lock`) was auto-generated by the build attempt and left uncommitted; finalize when the iOS Flutter build runs end-to-end.
- **podspec deployment target** is currently `:ios, '11.0'` while the example Runner is `13.0` — align to `13.0`.
