# Code of Conduct

## Our Standard

This project expects respectful, evidence-based collaboration.

Examples of expected behavior:

- Be direct, technical, and constructive
- Assume good faith unless there is clear evidence otherwise
- Focus criticism on code, design, data, and behavior, not people
- Share context, benchmarks, and reproductions when making strong claims
- Help keep discussions useful for future contributors

Examples of unacceptable behavior:

- Harassment, intimidation, or personal attacks
- Discriminatory or demeaning language
- Repeated bad-faith argumentation or deliberate derailment
- Publishing private information, credentials, or confidential data
- Spamming, trolling, or disrupting project workflows

## Scope

This code of conduct applies to repository issues, pull requests, code review, and other project communication spaces.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, or block participants whose behavior makes the project harder to maintain safely and productively.

If you need to report a conduct issue, contact the maintainers through a private channel. Public callout threads are discouraged unless immediate public moderation is necessary.
